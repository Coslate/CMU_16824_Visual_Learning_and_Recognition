        \item How does the test mAP and training loss change over time? Why do you think this is happening?

		\begin{solution}
			The test mAP increases very quickly at the beginning, then improves more slowly, and finally plateaus around a high value (roughly $0.79$) with only small fluctuations. The training loss shows the opposite trend: it drops sharply in the early stage, then continues decreasing more gradually, and eventually stabilizes at a low value.
			
			This behavior is expected. Since ResNet-18 is initialized with ImageNet pre-trained weights, the model already starts with useful visual features, so it can learn the PASCAL classification task much faster than a network trained from scratch. That is why we see a rapid early decrease in training loss and a rapid increase in test mAP. After this initial phase, the model has already captured most of the easy transferable structure, so additional gains become smaller and both curves begin to flatten.
		
			The learning rate schedule also helps explain the shape of the curves. When the learning rate is reduced, parameter updates become smaller, which usually makes training more stable and helps fine-tune the model near a local optimum. As a result, the training loss keeps decreasing slowly, while the test mAP improves only marginally and then saturates. The small oscillations are likely due to mini-batch SGD noise and the fact that mAP on the test set is a noisier evaluation metric than training loss.
		\end{solution}

			Both gradient histograms are centered close to zero throughout training, but they differ in spread and evolution. The histogram for \texttt{layer1.1.conv1.weight} is relatively narrower and more symmetric, which suggests that the gradients in this early convolutional layer are small and stable. In contrast, the histogram for \texttt{layer4.0.bn2.bias} is broader and appears to have a slightly longer tail, meaning the gradients in this deeper layer are generally larger and more variable.
			
			Over time, both histograms become more concentrated around zero. Early in training, the distributions are wider, indicating larger gradient updates. Later, they shrink and pile up closer to zero, showing that parameter updates are getting smaller as the model approaches convergence. This matches the behavior of the loss and mAP curves: once the network has learned most of the useful task-specific adjustments, fewer large updates are needed.
			
			A likely reason for the difference between the two layers is that the early convolutional layer is initialized from ImageNet pretraining and already contains useful low-level features such as edges and textures, so it only needs modest fine-tuning. The deeper batch-normalization bias is much closer to the classifier and is more task-specific, so it receives stronger gradients in order to adapt the higher-level representation to PASCAL classes. In addition, the learning rate decay also helps explain why both histograms narrow over time: smaller learning rates lead to smaller effective updates and hence gradients that cluster more tightly around zero.
		\end{solution}

However, I would not expect perfectly clean class boundaries because VOC is a multi-label dataset: many images contain multiple objects at once, so their features can lie between several semantic groups. In addition, t-SNE mainly preserves local neighborhoods rather than global distances, so partial clustering with some overlap is a realistic and reasonable outcome.
